# 수출 마케팅 광고 성과 대시보드 — 설계 스펙

## 개요

GNA(지엔에이)의 자회사 투고글로벌·투바이어의 자동차 수출 마케팅 광고 성과를 분석하는 웹 대시보드.

**목적:** 매체별 광고비 대비 성과를 빠르게 파악하여 예산 배분 최적화

## 기술 스택

- **Frontend:** Next.js 14 (App Router) + Tailwind CSS + shadcn/ui + Recharts
- **Backend/DB:** Supabase (PostgreSQL + Auth + RLS)
- **배포:** Vercel (외부 인원 접근 필요)

## 1. 페이지 구성

### 1-1. 대시보드 (메인)

**PC 기본 뷰 — 최근 7일 일별 합산:**
- 필터 바: 오늘 / 이번주 / 이번달 / 지난달 프리셋 + 날짜 범위 직접 선택 + 매체 필터
- 통합 테이블 (전체 매체 합산, 일별):
  - 컬럼: 날짜 | 콜량(전체/유효/무효/폐차) | 소진액 | 단가(전체기준/유효기준)
  - 오늘 행은 마지막 입력 시점 표시 (예: "09:30 기준")
  - 과거 날짜는 확정 데이터
  - 하단에 7일 합계 행
- 보조 차트 (테이블 아래):
  - 일별 콜량 추이 바 차트 (숫자 표기 포함)
  - 매체별 콜량 비중 파이 차트 (% + 건수 표기)

**PC 날짜 선택 시 — 매체별 합산 뷰:**
- 기본도 7일 일별 × 매체별 표시
- 날짜별로 매체 4개(네이버/당근/메타/구글) 행 + 소계 행
- 오늘/어제는 펼침, 나머지는 접힌 상태 (▸ 클릭 펼침)
- 보조 차트: 매체별 CPA 비교 바 차트 (단가 숫자 표기) + 매체별 비중 파이

**모바일 대시보드:**
- 일별 스와이프 카드 (7일, 좌우 스와이프)
  - 각 카드: 날짜, 콜량(유효 포함), 소진액, 폐차/부재/무효, CPA 전체/유효
  - 오늘 카드가 기본, 옆으로 스와이프하여 이전 날짜
  - 7개 인디케이터 점으로 위치 표시
- 아래에 7일 추이 차트 + 매체별 비중

**모바일 매체별 뷰:**
- 매체별 스와이프 카드 (네이버 → 당근 → 메타 → 구글)
- 각 카드에 해당 매체의 콜량 세부/소진액/CPA

### 1-2. 데이터 입력

**탭 구조:** 문의 관리 | 광고비 관리

**PC 레이아웃:** 좌측 컴팩트 입력폼 (300px) + 우측 이력+부재 처리

#### 문의 관리 탭

**입력 모드 2가지 (토글):**
1. **텍스트 작성 (기본):** textarea에 보고 텍스트 붙여넣기 or 직접 작성 → 파싱 → 미리보기 → 저장
2. **직접 입력 (서브):** 날짜/매체 선택 → 세부 내역 숫자 입력

**세부 내역 항목 (2열 배치):**
- 수출가능 (초록)
- 중고매입 (초록)
- 폐차 (노랑)
- 전화-네이버웹 추정 (초록)
- 부재 (파랑)
- 무효 (빨강) — 맨 마지막

**매체 드롭다운:** 네이버-홈페이지 / 네이버-랜딩 / 당근 / 메타 / 구글

**텍스트 파싱 규칙:**
- "투바이어 유입" 앞 매체 미표기 = 네이버 (기본)
- "ㅡㅡㅡ" 구분선 뒤 "당근" = 당근
- "메타" 키워드 = 메타
- (내수N) = 중고매입
- (폐차N) = 폐차
- (부재N) = 부재
- (무효N) = 무효
- 나머지 = 수출가능 (총건수 - 내수 - 폐차 - 부재 - 무효)
- 전화 건수도 파싱 → 전화(네이버웹 추정)으로 분류
- 채널 정보(전화/카톡/문자) 파싱 결과에 표시

**수출/중고매입 미분리 과거 데이터:**
- DB에 `valid_total` 필드로 저장 (수출/중고 각각은 null)
- 대시보드에 "유효(미분리)"로 표시

**우측 이력 테이블:**
- 해당 날짜의 매체별 입력 현황 (수출/중고/폐차/부재/무효/합계)
- 네이버/당근/메타/구글 + 합계 행

**부재 처리 (A방식):**
- 부재 미처리 건 리스트
- 드롭다운으로 상태 변경 (부재 → 수출가능/중고매입/폐차/무효)
- 변경 시 이력 자동 기록 (변경자·시간·이전→이후)
- 모바일: 왼쪽 스와이프로 상태 변경 버튼 노출

#### 광고비 관리 탭

**입력 모드 2가지 (토글):**
1. **텍스트 작성 (기본):** "네이버 소진액 : 96,432 원" 형식 파싱
2. **직접 입력 (서브):** 날짜/매체 선택 → 소진액 입력

**자동 계산:** 콜량 데이터 기반으로 건당 단가 (전체/유효) 자동 표시

**반복 입력:** "어제와 같은 매체로 입력" 버튼 — 매체 선택 건너뛰기

**우측 이력 테이블:**
- 매체별 소진액/콜량/단가(전체)/단가(유효)
- 미입력 매체는 "미입력" 표시
- 하단 총 소진액/평균 단가 요약

### 1-3. 사용자 관리 (관리자 전용)

**테이블:** 이메일 | 가입일 | 상태 | 역할 | 관리
- 이메일: 진한 파랑 볼드
- 가입일/역할: 진한 남색 볼드
- 상태: 승인(진한 초록) / 대기(진한 빨강)
- 대기 사용자: 역할 선택 드롭다운 + 승인 버튼

**역할 권한:**
| 역할 | 권한 |
|------|------|
| 콜량보고 | 대시보드 조회 + 문의 관리 탭 입력 |
| 소진액보고 | 대시보드 조회 + 광고비 관리 탭 입력 |
| 뷰어 | 대시보드 조회만 |
| 관리자 | 전체 접근 + 사용자 관리 |

### 1-4. 인증 (로그인/회원가입/비밀번호 찾기)

**로그인:** 이메일 + 비밀번호

**회원가입:**
- 이름
- 이메일: 영문 아이디 입력 + @ + 도메인 드롭다운 (naver.com / gmail.com / daum.net / kakao.com / hanmail.net / nate.com / 직접입력)
- 전화번호: 숫자만 입력 → 010-0000-0000 자동 포맷 (추후 문자 인증/본인인증 패스 연동)
- 비밀번호 (8자 이상)
- 비밀번호 확인
- 소속 (선택)
- 가입 신청 → 승인 대기 상태

**승인 대기 화면:** 가입 후 로그인 시 표시, 관리자 승인 전까지 접근 차단

**비밀번호 찾기:** 이메일 입력 → Supabase 기본 재설정 링크 발송

## 2. 데이터 구조

### 콜량 보고 타이밍
- 오전 08:00~13:00 접속 → "오전 보고" 자동 세팅 (전일 합계 + 당일 오전)
- 오후 17:00~23:00 접속 → "퇴근 보고" 자동 세팅 (당일 적립식)

### 데이터 표시 로직
- 과거 날짜 → 완성된 일 단위 합계
- 오늘 날짜 → 마지막 입력 시점까지의 누적값 + 시점 표시
- 일 단위 저장 + 입력 이력(보고 시점) 남김

### 콜량 세부 분류
- 유효: 수출가능 / 중고매입
- 폐차
- 부재 (통화 완료 후 다른 상태로 변경 가능, 이력 기록)
- 전화(네이버웹 추정) — 네이버 건수에 합산
- 무효: "문의한 적 없어요" 건만

### 광고 매체
- 네이버 (홈페이지 / 랜딩페이지 별도 구분)
- 당근
- 메타 (페이스북/인스타그램)
- 구글
- 추후 매체 추가 가능한 구조

## 3. 보안

- Supabase Auth: bcrypt 비밀번호 해싱, JWT 세션, 자동 만료/갱신
- Supabase RLS: 역할별 데이터 접근 제한
- API 키: .env 파일에 저장 (소스 노출 방지)
- Vercel 배포 시 자동 HTTPS/SSL
- 승인 안 된 유저 접근 차단 미들웨어
- Rate limiting: Supabase 기본 제공
